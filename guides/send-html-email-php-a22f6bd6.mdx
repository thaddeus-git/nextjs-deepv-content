# Send HTML Emails with Images in PHP

Sending HTML-formatted emails with embedded or linked images in PHP is a common requirement for applications like newsletters, notifications, and password resets. This guide will walk you through the process using the native `mail()` function and introduce more robust solutions like PHPMailer.

## Quick Answer

To send an HTML email using PHP's `mail()` function, you must set the `Content-Type` header to `text/html`. For images, link them using absolute URLs to an externally hosted server.

```php
<?php
$to = 'recipient@example.com';
$subject = 'HTML Email with Image';
$message = '
<html>
<head>
  <title>My HTML Email</title>
</head>
<body>
  <h1>Hello from PHP!</h1>
  <p>This is an HTML email with an image:</p>
  <img src="http://yourdomain.com/images/logo.png" alt="Company Logo">
  <p>Best regards,</p>
  <p>Your Team</p>
</body>
</html>';

$headers = "MIME-Version: 1.0\r\n";
$headers .= "Content-type:text/html;charset=UTF-8\r\n";
$headers .= 'From: Your Name <your_email@example.com>' . "\r\n";

if (mail($to, $subject, $message, $headers)) {
    echo "HTML email sent successfully!";
} else {
    echo "Failed to send HTML email.";
}
?>
```

## Ready-to-Use Code

Here are a few common scenarios for sending HTML emails.

### 1. Basic HTML Email with External Images

This is the most straightforward approach using PHP's built-in `mail()` function. Images are linked from a web server.

```php
<?php
$to = 'bob@example.com';
$subject = 'Website Change Request';

// Set headers for HTML email
$headers  = "From: Your Website <no-reply@yourdomain.com>\r\n";
$headers .= "Reply-To: support@yourdomain.com\r\n";
$headers .= "MIME-Version: 1.0\r\n";
$headers .= "Content-Type: text/html; charset=UTF-8\r\n";

// HTML message content
$message = '
<html>
<head>
  <title>Change Request</title>
</head>
<body>
  <p><strong>This is strong text</strong> while this is not.</p>
  <p>Here is an image from your server:</p>
  <img src="http://yourdomain.com/images/header_logo.png" alt="Logo" style="width:150px;">
  <p>Thank you for your request.</p>
</body>
</html>';

// Send the email
if (mail($to, $subject, $message, $headers)) {
    echo "Email sent successfully!";
} else {
    echo "Email sending failed.";
}
?>
```

This example demonstrates how to inject dynamic data into your HTML email message.

## Method 1: Using PHP's Native `mail()` Function

The `mail()` function is PHP's built-in way to send emails. While simple, it requires careful header construction for HTML content and doesn't directly support attachments or embedded images without complex MIME handling.

### Key Steps:

1.  **Set `Content-Type` Header**: This is crucial. You must specify `Content-Type: text/html; charset=UTF-8` in your email headers.
2.  **Use Absolute URLs for Images**: Email clients typically don't have access to your local file system. Images must be hosted on a publicly accessible web server, and their `src` attributes in the HTML should use full `http://` or `https://` URLs.
3.  **Inline CSS (Recommended)**: While you can link external CSS, many email clients strip or ignore it. For consistent rendering, it's best to use inline CSS styles directly within your HTML tags.

### Example: Basic HTML Email

```php
<?php
$to = 'recipient@example.com';
$subject = 'My First HTML Email';
$message = '
<html>
<head>
  <title>HTML Email</title>
  <style>
    body { font-family: Arial, sans-serif; line-height: 1.6; }
    .container { width: 80%; margin: 0 auto; padding: 20px; border: 1px solid #ddd; }
    h1 { color: #333; }
    p { color: #555; }
    img { max-width: 100%; height: auto; }
  </style>
</head>
<body>
  <div class="container">
    <h1>Welcome!</h1>
    <p>This is a <strong>rich HTML email</strong> sent from PHP.</p>
    <p>You can include paragraphs, lists, and even images:</p>
    <img src="https://via.placeholder.com/300x150.png?text=Your+Logo" alt="Placeholder Logo">
    <p>Visit our website: <a href="http://www.example.com">Example.com</a></p>
  </div>
</body>
</html>';

// Always set content-type when sending HTML email
$headers = "MIME-Version: 1.0\r\n";
$headers .= "Content-type:text/html;charset=UTF-8\r\n";

// More headers
$headers .= 'From: Webmaster <webmaster@example.com>' . "\r\n";
$headers .= 'Cc: myboss@example.com' . "\r\n";

if (mail($to, $subject, $message, $headers)) {
    echo "HTML email sent successfully!";
} else {
    echo "Email sending failed.";
}
?>
```

## Method 2: Using PHPMailer for Advanced Features

For more robust email sending, including file attachments, embedded images (CID), SMTP authentication, and better error handling, using a dedicated library like PHPMailer is highly recommended.

### Why PHPMailer?

*   **Attachments**: Easily attach files from your server.
*   **Embedded Images**: Embed images directly into the email body using Content-ID (CID), so they appear even if the recipient is offline or the external server is down.
*   **SMTP Support**: Send emails via an external SMTP server (e.g., Gmail, SendGrid, Mailgun) for higher deliverability and authentication.
*   **HTML & Plain Text**: Automatically generates a plain-text alternative for HTML emails.
*   **Error Handling**: Provides more detailed error messages than `mail()`.

### Installation

You can install PHPMailer via Composer:

```bash
composer require phpmailer/phpmailer
```

### Example: HTML Email with Embedded Image and Attachment

```php
<?php
use PHPMailer\PHPMailer\PHPMailer;
use PHPMailer\PHPMailer\Exception;

require 'vendor/autoload.php'; // Adjust path if not using Composer

$mail = new PHPMailer(true); // Pass `true` to enable exceptions

try {
    // Server settings
    $mail->isSMTP();                                            // Send using SMTP
    $mail->Host       = 'smtp.example.com';                     // Set the SMTP server to send through
    $mail->SMTPAuth   = true;                                   // Enable SMTP authentication
    $mail->Username   = 'your_email@example.com';               // SMTP username
    $mail->Password   = 'your_smtp_password';                   // SMTP password
    $mail->SMTPSecure = PHPMailer::ENCRYPTION_STARTTLS;         // Enable TLS encryption; `PHPMailer::ENCRYPTION_SMTPS` also accepted
    $mail->Port       = 587;                                    // TCP port to connect to

    // Recipients
    $mail->setFrom('from@example.com', 'Your Name');
    $mail->addAddress('recipient@example.com', 'Recipient Name'); // Add a recipient
    $mail->addReplyTo('info@example.com', 'Information');
    $mail->addCC('cc@example.com');
    $mail->addBCC('bcc@example.com');

    // Attachments
    // Create a dummy file for attachment if it doesn't exist
    if (!file_exists('attachment.pdf')) {
        file_put_contents('attachment.pdf', 'This is a dummy PDF attachment.');
    }
    $mail->addAttachment('attachment.pdf');         // Add attachments
    
    // Create a dummy image for embedding if it doesn't exist
    if (!file_exists('logo.png')) {
        // You'd typically have a real image here. For demonstration, let's create a tiny one.
        $img = imagecreatetruecolor(100, 50);
        $white = imagecolorallocate($img, 255, 255, 255);
        $blue = imagecolorallocate($img, 0, 0, 255);
        imagefill($img, 0, 0, $white);
        imagestring($img, 5, 10, 15, 'Logo', $blue);
        imagepng($img, 'logo.png');
        imagedestroy($img);
    }
    $mail->AddEmbeddedImage('logo.png', 'logo_cid', 'logo.png'); // Embed an image

    // Content
    $mail->isHTML(true);                                  // Set email format to HTML
    $mail->Subject = 'PHPMailer HTML Email with Embedded Image and Attachment';
    $mail->Body    = '
    <html>
    <head>
      <title>PHPMailer Test</title>
    </head>
    <body>
      <h1>Hello from PHPMailer!</h1>
      <p>This email contains an <strong>embedded image</strong> and an <strong>attachment</strong>.</p>
      <img src="cid:logo_cid" alt="Embedded Logo" style="width:100px;">
      <p>Find your attached document below.</p>
      <p>Best regards,</p>
      <p>Your PHPMailer App</p>
    </body>
    </html>';
    $mail->AltBody = 'This is the plain text message body for non-HTML mail clients. It also contains an attachment.';

    $mail->send();
    echo 'Message has been sent';
} catch (Exception $e) {
    echo "Message could not be sent. Mailer Error: {$mail->ErrorInfo}";
}
?>
```

## Common Problems & Solutions

### 1. Images Not Displaying

*   **Problem**: Images appear as broken links or don't load.
*   **Solution (Native `mail()`):**
    *   Ensure image `src` attributes use **absolute URLs** (e.g., `http://yourdomain.com/images/logo.png`). Relative paths will not work.
    *   Verify the image file exists at the specified URL and is publicly accessible.
    *   Check for typos in the URL.
*   **Solution (PHPMailer):**
    *   If using `AddEmbeddedImage()`, ensure the `cid` in the `src="cid:..."` matches the one provided to `AddEmbeddedImage()`.
    *   Verify the path to the image file on your server is correct.

### 2. HTML Not Rendering Correctly

*   **Problem**: Email client displays raw HTML tags or renders the layout incorrectly.
*   **Solution**:
    *   **`Content-Type` Header**: Double-check that your headers include `Content-Type: text/html; charset=UTF-8` (or `iso-8859-1`). This is the most common cause.
    *   **Inline CSS**: Many email clients have limited CSS support and often strip external stylesheets. Use inline styles (`<p style="color: blue;">`) for critical styling.
    *   **HTML Structure**: Keep your HTML simple and use tables for layout, as they are more consistently supported across email clients than modern CSS layouts (like Flexbox or Grid). Avoid `<html>`, `<head>`, `<body>` tags within the `$message` variable if using `mail()`, as some webmail clients might wrap your content in their own, leading to nested/invalid HTML. PHPMailer handles this better.
    *   **Email Client Differences**: Test your HTML email across various email clients (Gmail, Outlook, Apple Mail, etc.) as they all render HTML differently. Tools like Litmus or Email on Acid can help.

### 3. Emails Going to Spam

*   **Problem**: Your HTML emails are frequently marked as spam.
*   **Solution**:
    *   **Sender Authentication**: Use an authenticated SMTP server (e.g., via PHPMailer) rather than the default `mail()` function, which often sends from `localhost` or an unverified domain.
    *   **SPF, DKIM, DMARC**: Configure these DNS records for your sending domain. They help email providers verify that your emails are legitimate.
    *   **Content Quality**: Avoid spammy keywords, excessive images, or links to suspicious domains.
    *   **Plain Text Alternative**: PHPMailer automatically generates a plain-text version (`AltBody`), which is good for deliverability. If using `mail()`, you'd need to manually create a multipart email.

### 4. Character Encoding Issues

*   **Problem**: Special characters (e.g., accents, emojis) appear as garbled text.
*   **Solution**:
    *   Ensure your `Content-Type` header specifies `charset=UTF-8`.
    *   Make sure your PHP script file itself is saved with UTF-8 encoding.
    *   If using PHPMailer, it defaults to UTF-8, but ensure your database connection (if pulling content from there) also uses UTF-8.

## Real-World Use Cases

*   **E-commerce Order Confirmations**: Send visually appealing order summaries with product images, pricing, and shipping details.
*   **Newsletter Campaigns**: Distribute rich, branded newsletters with articles, promotions, and calls to action.
*   **Password Reset Emails**: Provide a clear, branded email with a link to reset a password.

## Related: Other PHP Email Operations

*   **Sending Plain Text Emails**: Simply omit the `Content-Type: text/html` header.
*   **Sending Emails with Attachments (without PHPMailer)**: This is significantly more complex with `mail()`, requiring manual construction of multipart MIME messages. It's generally not recommended for production.
*   **Using Templating Engines**: For complex HTML emails, consider using a templating engine like Twig or Blade to separate your HTML structure from your PHP logic.
*   **Transactional Email Services**: For high-volume or critical emails, integrate with services like SendGrid, Mailgun, or Amazon SES, which offer APIs and handle deliverability, tracking, and scaling.

## Summary

Sending HTML emails in PHP is essential for modern web applications. While the native `mail()` function can handle basic HTML emails by setting the `Content-Type` header and linking external images, it quickly becomes cumbersome for more advanced needs like attachments, embedded images, or reliable SMTP delivery. For these scenarios, libraries like PHPMailer provide a robust, feature-rich, and easier-to-manage solution, significantly improving deliverability and developer experience. Always test your HTML emails across various clients to ensure consistent rendering.
