```csharp
Hello, {name}!

This is a multiline string.
```


## Method 1: Raw String Literals (C# 11+)

Raw string literals, introduced in C# 11, provide the most elegant solution for multiline strings, especially when dealing with content that requires precise formatting, such as JSON, XML, or SQL queries. They start and end with at least three double-quote characters (`"""`) and allow you to embed any characters, including newlines and quotes, without escaping. The indentation of the string is determined by the closing delimiter.

### Persona: üèóÔ∏è Architecture Builder, üé® Output Focused

**üèóÔ∏è Architecture Builders** will appreciate the clean syntax for defining complex data structures like JSON or XML directly in code, improving maintainability and reducing parsing errors. **üé® Output Focused** users will love the exact control over whitespace and formatting, ensuring the output matches specifications precisely.

### How it Works

1.  **Delimiter:** Start and end the string with at least three double-quotes (`"""`). If your string contains three consecutive double-quotes, you'll need to use four (or more) for the delimiter.
2.  **Indentation:** The indentation of the closing delimiter (`"""`) determines the minimum indentation for the entire string. Any whitespace to the left of the closing delimiter is removed from every line in the raw string literal. This allows you to indent your code for readability without affecting the string's content.
3.  **Interpolation:** Raw string literals can be interpolated by adding a `$` prefix, similar to interpolated strings. If your string contains `{` or `}` characters that are not part of an interpolation, you can add more `$` characters to the prefix (e.g., `$$"""`) and match the number of braces in your interpolation (e.g., `{{variable}}`).

### Examples

## Method 2: Interpolated Verbatim String Literals (C# 8.0+)

Introduced in C# 6.0 (interpolation) and combined with verbatim strings in C# 8.0, this method offers a powerful and highly readable way to create multiline strings that include embedded expressions. It combines the `$` for interpolation and `@` for verbatim string literals, resulting in `$@""` or `@$""`.

### Persona: üöÄ Speed Seeker, üìö Learning Explorer

**üöÄ Speed Seekers** will find this method incredibly efficient for quickly constructing complex strings with dynamic content. **üìö Learning Explorers** will appreciate how it clearly demonstrates modern C# features, making code both powerful and easy to understand.

### How it Works

1.  **Prefixes:** Start the string with both `$` and `@` (e.g., `$@"..."` or `@$"..."`). The order doesn't matter, but `$@` is more common.
2.  **Multiline Support:** The `@` prefix allows the string to span multiple lines, preserving newlines and most whitespace directly as typed.
3.  **Interpolation:** The `$` prefix enables embedding C# expressions directly within the string using curly braces `{}`.
4.  **Escaping:** Double quotes within the string must be escaped by doubling them (`""`). Backslashes (`\`) are treated as literal characters, not escape sequences.

### Examples

#### Basic Interpolated Verbatim String

```csharp

You are cordially invited to the
{eventName}
on {eventDate:D} at {location}.

We look forward to seeing you there!
";

Console.WriteLine("--- Example 2.1: Basic Interpolated Verbatim String ---");
Console.WriteLine(invitation);
// Expected Output (date format depends on culture):

// Tech Summit 2025
```

## Method 3: Verbatim String Literals (C# 1.0+)

Verbatim string literals, denoted by the `@` prefix, have been a part of C# since its inception. They allow a string to span multiple lines and treat backslash characters (`\`) as literal characters rather than escape sequences. This makes them ideal for paths, regular expressions, and simple multiline text where interpolation isn't needed.

### Persona: üîß Problem Solver, ‚ö° Legacy Maintainer

**üîß Problem Solvers** will find this a reliable "just works" solution for basic multiline strings without the complexity of interpolation. **‚ö° Legacy Maintainers** will frequently encounter and use this method in older codebases, as it's been available since C# 1.0.

### How it Works

1.  **Prefix:** Start the string with an `@` symbol immediately followed by a double quote (`@"`).
2.  **Multiline Support:** The string can span multiple lines, and all newlines and whitespace (including leading whitespace on subsequent lines) are preserved as part of the string content.
3.  **No Escape Sequences:** Backslashes (`\`) are treated as literal characters. For example, `@"\n"` is a literal backslash followed by 'n', not a newline character.
4.  **Escaping Double Quotes:** To include a double quote within a verbatim string, you must double it (`""`).

#### Verbatim String with `String.Trim()` for Indentation Control

```csharp
// Example 3.5: Verbatim String with String.Trim() for indentation control
// Verbatim strings preserve all whitespace, including leading whitespace.
// To remove unwanted leading indentation, you can use Trim().
string indentedText = @"
    This line is indented.
        This line is more indented.
    And this line is also indented.
";

Console.WriteLine("\n--- Example 3.5: Verbatim String with Trim() (Original) ---");

// To remove the common leading whitespace, you can use TrimStart() or a custom trim.
// For more advanced trimming, consider String.Replace() or regex.

Console.WriteLine("\n--- Example 3.5: Verbatim String with Trim() (Trimmed) ---");
Console.WriteLine(trimmedText);

// For more precise control over leading indentation, especially for code blocks:
// You might need to manually remove common leading whitespace or use a utility function.
// Example of a simple custom trim for common leading whitespace:
string[] lines = indentedText.Split(new[] { Environment.NewLine }, StringSplitOptions.None);
int minIndent = int.MaxValue;
foreach (string line in lines)
{
    if (string.IsNullOrWhiteSpace(line)) continue;
    int currentIndent = line.TakeWhile(char.IsWhiteSpace).Count();
    if (currentIndent < minIndent)
    {
        minIndent = currentIndent;
    }
}

string cleanedText = string.Join(Environment.NewLine, lines.Select(line =>
    line.Length >= minIndent ? line.Substring(minIndent) : line));

Console.WriteLine("\n--- Example 3.5: Verbatim String with Custom Indent Trim ---");
Console.WriteLine(cleanedText);
// Expected Output (trimmed):
// This line is indented.
//     This line is more indented.
// And this line is also indented.
```

## Method 4: String Concatenation with Newline Characters (C# 1.0+)

This is the most basic and oldest method for creating multiline strings in C#. It involves concatenating multiple string literals or variables using the `+` operator and explicitly inserting newline characters (`\n`, `\r\n`, or `Environment.NewLine`).

### Persona: ‚ö° Legacy Maintainer

**‚ö° Legacy Maintainers** will be very familiar with this approach, as it was the primary way to construct multiline strings before verbatim and interpolated strings became common. It's often found in older codebases.

### How it Works

1.  **Concatenation:** Use the `+` operator to join individual string segments.
2.  **Newline Characters:**
    *   `\n`: Represents a line feed (LF). Common on Unix-like systems.
    *   `\r\n`: Represents a carriage return followed by a line feed (CRLF). Standard on Windows.
    *   `Environment.NewLine`: A platform-agnostic way to get the correct newline sequence for the current operating system. This is generally preferred for portability.
3.  **Escaping:** Standard string literal rules apply, meaning backslashes (`\`) are escape characters (e.g., `\\` for a literal backslash, `\"` for a literal double quote).

### Examples

#### Basic Concatenation with `\n`

```csharp
// Example 4.1: Basic Concatenation with \n

                + "Line 2\n"
                + "Line 3.";

Console.WriteLine("--- Example 4.1: Basic Concatenation with \\n ---");
Console.WriteLine(message1);
// Expected Output:
// Line 1
// Line 2
// Line 3.
```

#### Concatenation with `Environment.NewLine`

```csharp
// Example 4.2: Concatenation with Environment.NewLine for portability
```

## Method 5: String.Join with Arrays (C# 1.0+)

The `String.Join` method provides a clean and efficient way to concatenate elements of a string array or any `IEnumerable<string>` into a single string, using a specified separator. This is particularly useful when you have a collection of lines that you want to combine into a multiline string.

### Persona: üìö Learning Explorer, üèóÔ∏è Architecture Builder

**üìö Learning Explorers** will find this method demonstrates a functional programming style for string manipulation. **üèóÔ∏è Architecture Builders** might use this for dynamically generating configuration files or reports where lines are constructed programmatically and then joined.

### How it Works

1.  **`String.Join(separator, values)`:** Takes a separator string (e.g., `Environment.NewLine`) and an array or enumerable of strings.
2.  **Efficiency:** It's generally more efficient than repeated `+` concatenation for a large number of strings because it allocates memory only once for the final string.
3.  **Readability:** Can improve readability when dealing with a list of distinct lines.

### Examples

#### Basic `String.Join` with `Environment.NewLine`

```csharp
// Example 5.1: Basic String.Join with Environment.NewLine
string[] lines = new string[]
{
    "First line of text.",
    "Second line, dynamically generated.",
    "Third and final line."
};

string multilineText = string.Join(Environment.NewLine, lines);

Console.WriteLine("--- Example 5.1: Basic String.Join ---");
Console.WriteLine(multilineText);
// Expected Output:
// First line of text.
// Second line, dynamically generated.
// Third and final line.
```

#### `String.Join` with a List of Strings

```csharp
// Example 5.2: String.Join with a List<string>
List<string> shoppingList = new List<string>
{
    "Milk",
    "Eggs",
    "Bread",
    "Cheese"
};

string formattedList = "Shopping List:" + Environment.NewLine
                     + string.Join(Environment.NewLine, shoppingList.Select(item => $"- {item}"));

Console.WriteLine("\n--- Example 5.2: String.Join with a List<string> ---");
Console.WriteLine(formattedList);
// Expected Output:
// Shopping List:
// - Milk
// - Eggs
// - Bread
// - Cheese
```

#### Generating a CSV Header with `String.Join`

```csharp

string csvHeaderLine = string.Join(",", csvHeaders);

Console.WriteLine("\n--- Example 5.3: Generating a CSV Header ---");
Console.WriteLine(csvHeaderLine);
// Expected Output:
// ID,Name,Email,Status
```

#### `String.Join` for a Multiline SQL `IN` Clause

```csharp

SELECT
    ProductName,
    Price

//     ProductName,
//     Price
// FROM
//     Products
// WHERE
//     ProductID IN (101, 105, 203, 210);
```

#### Combining `String.Join` with LINQ for Dynamic Content


## Method 6: StringBuilder for Dynamic Multiline Strings (C# 1.0+)

When you need to construct a multiline string dynamically, especially within loops or conditional logic, and performance is a concern (e.g., building very large strings), `System.Text.StringBuilder` is the go-to solution. It avoids the overhead of creating many intermediate string objects that occur with repeated string concatenation.

### Persona: üèóÔ∏è Architecture Builder, üöÄ Speed Seeker

**üèóÔ∏è Architecture Builders** will use `StringBuilder` for robust, scalable solutions where strings are built incrementally. **üöÄ Speed Seekers** will opt for `StringBuilder` when performance is critical, particularly for large or frequently constructed strings.

### How it Works

1.  **Mutable String:** `StringBuilder` is a mutable sequence of characters, unlike `string` which is immutable. This means operations like `Append` or `AppendLine` modify the existing `StringBuilder` object rather than creating new string objects.
2.  **`Append` and `AppendLine`:**
    *   `Append(string value)`: Adds the specified string to the end of the `StringBuilder`.
    *   `AppendLine(string value)`: Adds the specified string followed by a newline character (`Environment.NewLine`). There's also an overload `AppendLine()` that just adds a newline.
3.  **`ToString()`:** Once all parts of the string have been added, call `ToString()` to convert the `StringBuilder` content into an immutable `string` object.
4.  **Capacity:** You can optionally specify an initial capacity to minimize reallocations, though `StringBuilder` handles resizing efficiently.

### Examples

#### Basic `StringBuilder` Usage

```csharp
// Example 6.1: Basic StringBuilder for a simple multiline message
System.Text.StringBuilder sb = new System.Text.StringBuilder();

sb.AppendLine("This is the first line.");
sb.Append("This is the second line, ");
sb.AppendLine("and it continues here.");
sb.AppendLine("Finally, the third line.");

string result = sb.ToString();

Console.WriteLine("--- Example 6.1: Basic StringBuilder Usage ---");
Console.WriteLine(result);
// Expected Output:
// This is the first line.
// This is the second line, and it continues here.
// Finally, the third line.
```

#### Building a Report with `StringBuilder` in a Loop

```csharp
// Example 6.2: Building a Report with StringBuilder in a Loop

{
```
